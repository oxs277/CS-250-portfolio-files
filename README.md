CS 250 Module Eight Journal Reflection

How do I interpret user needs and implement them into a program? How does creating user stories help with this?

I interpret user needs by looking at what the user is trying to accomplish and turning those needs into clear software requirements. User stories help with this because they explain the feature from the user’s point of view. Instead of only thinking about the technical side, user stories help the team understand who the feature is for, what the user needs, and why it matters. This makes it easier to design a program that actually solves the user’s problem.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach developing programs by first understanding the requirements, then breaking the work into smaller steps. After that, I can plan, build, test, and revise the program as needed. Agile processes are helpful because they allow changes to be made throughout the project instead of waiting until the end. In the future, I hope to use Sprint Planning, Daily Scrums, Sprint Reviews, and Retrospectives because they support communication, feedback, and continuous improvement.

What does it mean to be a good team member in software development?

Being a good team member in software development means communicating clearly, completing assigned work, helping others when needed, and being open to feedback. Software development is rarely done alone, so teamwork is important for building a successful product. A good team member also stays flexible when requirements change and focuses on the overall success of the project instead of only their own tasks.
